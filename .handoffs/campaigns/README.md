# Campaigns

Campaigns are bigger than one handoff. Use them for programs that fan out across many repos, many tasks, or several review layers.

Current campaigns:

- [ecosystem-health-audit](ecosystem-health-audit/README.md): 5-phase audit covering contract integrity, code quality, architecture drift, bug patterns, and inter-tool interaction — 2 passes per phase — started 2026-04-22
- [external-audit-gap-map](external-audit-gap-map/README.md): maps the external audit synthesis to the active handoff queue and calls out missing handoffs or campaign-sized gaps
- [orchestration-reset](orchestration-reset/README.md): turns the orchestration reset into a sequenced campaign with one authority model, one runtime vocabulary, and one child-handoff queue
- [operator-runtime-follow-ons](operator-runtime-follow-ons/README.md): organizes the worthwhile secondary audit findings into runtime hardening, session identity, operator surfaces, and host-readiness tracks

Archived campaigns live under [../archive/campaigns/README.md](/Users/williamnewton/projects/basidiocarp/.handoffs/archive/campaigns/README.md).

If the work can be expressed as one actionable handoff with one verification script, it does not belong here.
