# Session Notes

Session notes are short resume documents. They capture what happened in one work session, what matters next, and what to reopen first.

## Layout

- `open/`: current notes that still matter
- `archive/YYYY/MM/`: old notes kept for lookup

## Rules

- Name files as `YYYY-MM-DD-topic.md`.
- Keep them short. They should help a later session resume, not replace project documentation.
- Move notes out of `open/` once the work is done or superseded.
